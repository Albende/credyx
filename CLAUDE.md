## What this project is

**CreditLens** is a B2B credit intelligence platform. The goal: given a
company name or identifier, retrieve real registry data + filed balance
sheets from official government sources, then run a credit risk analysis
via Gemini (through kie.ai) and return a structured `RiskAssessment`
(score 0–100, recommendation APPROVE/REVIEW/REJECT, recommended credit
limit in EUR, key signals, red flags, confidence).

Stack: Next.js 15 + Tailwind on the frontend; FastAPI + Pydantic v2 +
SQLAlchemy 2 (async) + Postgres 16 + Redis 7 on the backend. Adapters per
country live under `packages/adapters/{cc}/`. The risk engine
(`packages/risk`) pre-computes deterministic financial ratios before
calling the LLM (`packages/llm`) — the model never does arithmetic.

Currently MVP, no auth.

## Non-negotiable rules (do not relax without asking)

1. **No mock data anywhere.** If an adapter can't return real data, it
   raises `AdapterNotImplementedError`. The API translates that to
   `501 not_implemented`. Never invent numbers, names, dates, or PDFs.
2. **No paid commercial APIs in MVP** (no Creditsafe, D&B, Bureau van Dijk,
   InfoCamere, Lursoft, FinStat, Bolagsverket paid tier, etc.). Free
   government registries + free aggregators (GLEIF, OpenCorporates free
   tier, OpenSanctions) only. Paid integration is a Phase-2 decision.
3. **One LLM provider class.** Everything model-related goes through
   `packages.llm.providers.LLMProvider`. `KieAIGeminiProvider` is the only
   impl today. If you swap it, keep the interface stable.
4. **Respect rate limits + robots.txt.** Adapters throttle to documented
   per-source limits. `packages.adapters._base.http.get_with_retry` honors
   `Retry-After` headers. If a registry blocks (CAPTCHA, geoblock,
   paywall), document it in `docs/countries/{cc}.md` and mark the adapter
   blocked — don't keep retrying.
5. **Deterministic ratios first, model second.** `packages.risk.ratios`
   computes current/quick/D-E/ROE/ROA/Altman-Z/etc. The LLM gets these as
   named context. Don't ask the LLM for math.
6. **Cache aggressively in Postgres.** Registry data 7d TTL, filings 30d.
   `?force_refresh=true` available on every fetch endpoint. Risk
   assessments are persisted forever for audit.
7. **Type-safe contracts.** `packages/shared/models.py` is the single
   source of truth for shapes. Pydantic v2 everywhere on the backend; the
   TS types in `apps/web/src/lib/api.ts` mirror them by hand for now.

## Key commands

```bash
# Full local stack (Postgres, Redis, API, web, pgAdmin)
docker compose up --build

# Backend only (with deps installed)
PYTHONPATH=. uvicorn apps.api.app.main:app --reload

# Run real-API validation (writes docs/VALIDATION_REPORT.md)
python scripts/validate.py

# Unit + integration tests (integration hits real registries — slow)
PYTHONPATH=. pytest                          # all
PYTHONPATH=. pytest -m "not integration"     # unit only
PYTHONPATH=. pytest packages/adapters/us     # one adapter

# Frontend
cd apps/web && npm install && npm run dev
```

## Adding a new country adapter

Worked example, ~1 hour for a country with a free JSON API:

1. `mkdir packages/adapters/{cc}` and add `__init__.py`, `adapter.py`.
2. Read `docs/countries/{cc}.md` for the registry URL, identifier format,
   auth requirements, and test companies.
3. Subclass `CountryAdapter` (`packages/adapters/_base/adapter.py`):
   - Set `country_code`, `country_name`, `identifier_types`,
     `primary_identifier`, `rate_limit_per_minute`.
   - If the source needs a key, set `requires_api_key = True` and
     `api_key_env = "..."`.
   - Implement `search_by_name`, `lookup_by_identifier`,
     `fetch_financials`. Use `build_http_client` from `_base/http.py`.
4. Register the adapter in `packages/adapters/registry.py` →
   `_build_real_adapters`.
5. Add an integration test in `packages/adapters/{cc}/tests/test_{cc}.py`
   marked `@pytest.mark.integration`. **Tests must hit the real source.**
   Use one of the real test companies from the country doc.
6. Update `docs/countries/{cc}.md` status block.
7. Re-run `python scripts/validate.py`.

## Style and conventions

- Python 3.11+. Type hints everywhere. `from __future__ import annotations`
  at the top of modules.
- No defensive try/except around things that can't fail. Catch at module
  boundaries (HTTP, parsing) where it matters.
- No comments that explain *what* — the names should. Only write comments
  for *why* and only when non-obvious. No backwards-compat shims, no
  removed-code comments.
- One blank line between top-level functions, two between classes
  (default formatting).
- Adapters return Pydantic models, never raw dicts.
- Times: timezone-naive `datetime.utcnow()` is used today; if you migrate,
  switch the whole codebase together.

## What's left to cover all world companies

The MVP covers 8 live countries (UK, US, FR, NL, CZ, EE, NO, FI) and stubs
23 European + TR. To get to **global coverage**, here's the realistic
roadmap, in priority order.

### Cross-cutting infrastructure work (parallel to country adapters)

Without these, country breadth alone hits a ceiling.

1. **PDF text extraction pipeline.** `pypdf` is in `requirements.txt` but
   not yet wired. For UK / DE / FR / IT / ES / NO filings that are PDF-only,
   extract text in a Celery worker, store the text, pass excerpts to the
   LLM via `pdf_text_excerpts={year: text}` (already supported by
   `LLMService.analyze_credit_risk`).
2. **ESEF (European Single Electronic Format) XBRL parser.** All EU
   listed-company filings from 2021+ must be in iXBRL ESEF taxonomy.
   Build one parser at `packages/risk/xbrl_esef.py` and re-use across
   every EU adapter.
3. **Browser pool + proxy rotation.** `packages/adapters/_base/browser.py`
   with a Playwright async context-manager. Proxy interface so paid
   providers can be swapped in.
4. **Document object storage.** Filings can run to hundreds of MB per
   company. Add S3 (MinIO locally) for raw doc storage; keep only URLs and
   structured JSON in Postgres.
5. **Currency normalization.** Right now `currency` is stored as-is.
   Centralize FX conversion to EUR (ECB reference rates, free daily feed)
   so the risk engine can reason in one unit.
6. **Industry benchmarks.** Altman-Z thresholds and "good" ratio bands
   differ massively by industry. Build a lookup table per NACE 2-digit
   code so the prompt can include "for this industry, median current
   ratio is X".
7. **Change detection / monitoring.** Subscribe to registry change feeds
   where available (UK Companies House has webhooks; SEC EDGAR has
   accession-feed RSS). Push deltas into Kafka or just a webhook out.
8. **Sanctions / PEP integration.** OpenSanctions client exists but is
   not yet wired into the risk pipeline. Should be: `risk.engine` calls
   `OpenSanctionsClient.screen()` and surfaces hits as automatic red
   flags before the LLM runs.
9. **Web admin / job dashboard.** Today `ingestion_jobs` is queryable via
   `/api/jobs/{id}`. A simple admin page listing recent jobs, failures,
   re-run buttons would help operations.

### Realistic effort to "all world companies"

- **Full Europe (EU27 + UK + EFTA + Balkans)**: 8–12 weeks engineering.
- **+ Türkiye + CIS**: +2–4 weeks.
- **+ Asia-Pacific (top 14)**: +8–12 weeks (CN and IN dominate the work).
- **+ Americas**: +4–6 weeks.
- **+ MENA / Africa**: +6–8 weeks.
- **Cross-cutting infra (PDF, ESEF, proxy, FX, storage, sanctions wiring)**: ~3–4 weeks overlapping with the above.

**Total to genuinely global**: roughly **9–12 months of one senior
engineer's time**, or **3–4 months for a small focused team**. The
codebase is structured so countries can be added one at a time without
touching the core — the boring path forward.

### What to NOT do, even when tempted

- **Don't switch the LLM call out of `packages/llm`.** If you need
  function-calling, vision, or thinking-mode, extend the provider
  interface, don't introduce a side-channel call.
- **Don't add "default fallback" mock data when an adapter fails.** A 501
  surfaces the real state; a 200 with fake numbers is silently dangerous
  to a credit decision.
- **Don't combine adapter logic with route logic.** Adapters return DTOs;
  routes coordinate. Mixing them makes per-country swaps painful.
- **Don't bypass the rate limiter for "admin" or "internal" calls.** Use
  a separate Redis namespace if you genuinely need a higher tier — never
  remove the limit.
